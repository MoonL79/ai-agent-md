# AI Agent MD

这个目录用于维护 Codex 与 Claude 协作时的基础规范。

## 文件说明

- `CLAUDE.md`
  - 标准开发规范。
  - 这是跨项目通用的基础规范。
  - 当项目自身没有额外规范时，默认遵循该文件。

- `COMMANDER_MODE.md`
  - Codex 的工作模式说明。
  - 定义 Codex 作为主指挥官、Claude 作为默认执行者时的职责边界、调度方式、介入条件和升级策略。

## 使用顺序

1. 先读取 `CLAUDE.md`，明确通用开发规范。
2. 再读取 `COMMANDER_MODE.md`，明确协作方式与调度规则。
3. 进入具体项目后，再读取该项目根目录下的 `CLAUDE.md`。

## 规范优先级

- 通用规范来自本目录的 `CLAUDE.md`
- 协作模式来自本目录的 `COMMANDER_MODE.md`
- 项目规范来自当前项目根目录的 `CLAUDE.md`

执行任务时：

- 如果项目根目录存在 `CLAUDE.md`，则同时遵守通用规范和项目规范。
- 如果项目根目录不存在 `CLAUDE.md`，则只遵守本目录中的通用规范。

## 默认协作关系

- Codex 负责宏观分析、任务拆分、调度、监控和纠偏。
- Claude 负责默认执行。
- 若 Claude 长时间无进展、反复失败或方向错误，Codex 先尝试恢复 Claude 的执行能力。
- 仅在多次介入后仍无法推进时，Codex 才亲自执行。
